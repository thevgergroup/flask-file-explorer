# Contributing to Flask File Explorer

Thank you for considering contributing to Flask File Explorer! We welcome contributions from the community to help improve the project.

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing to this project.

## How to Report Issues

If you encounter any issues or bugs, please report them using the [GitHub Issues](https://github.com/thevgergroup/flask-file-explorer/issues) feature. Provide as much detail as possible to help us diagnose and fix the problem quickly.

## How to Submit Pull Requests

1. Fork the repository and clone your fork.
2. Create a new branch for your changes: `git checkout -b feature/your-feature-name`.
3. Make your changes and commit them with clear and descriptive messages.
4. Push your changes to your forked repository.
5. Open a Pull Request against the `main` branch of the original repository. Provide a clear description of your changes and any additional context necessary for reviewers.

## Coding Standards

- Follow [PEP 8](https://pep8.org/) coding style guidelines.
- Maintain clear and concise comments in your code.
- Ensure your code is well-documented.

## Testing Guidelines

- Ensure all existing tests pass before submitting your changes.
- Write new tests for any new features or bug fixes.
- Use `pytest` for running tests.

## Contact Information

For any questions or additional guidance, feel free to reach out via [email](mailto:example@example.com) or open an issue on GitHub to discuss.
